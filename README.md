#### 2. RESUMEN DEL PROYECTO, METAS Y OBJETIVOS
- **Resumen:** Keep-On es una herramienta dirigida a los estudiantes del estudio técnico en computación, el cual permite a los alumnos regularizarse y/o apoyarse para poder tener una buena estancia en la misma y mantener un buen contacto entre alumno y profesor.
    
- **Metas:**
	- Lograr que los alumnos puedan tener un mejor desempeño en la ETE y, a su vez, reducir la tasa de deserción. 
  - Establecer una mejor interacción alumno - profesor
  - Lograr la utilización regular de la página y posicionar su funcionalidad como un elemento clave para la prevalencia del estudio técnico.

- **Objetivos:**
  - Apoyar al estudiantado del ETE en computación en el mantenimiento su rendimiento académico.
  - Disminuir la deserción. 
  - Establecer un contacto más cercano con el profesor encargado.
  - Lograr estandarizar la página como medio para resolver las problemáticas de los alumnos.

#### 3. PÚBLICO OBJETIVO (UX)
Alumnos del primer año del ETE en computación del plantel 6 “ANTONIO CASO” y profesores asignados a la enseñanza del mismo.

#### 4. PROPÓSITO Y ALCANCE
- **En alcance (Entregables):**
	- Una página de inicio
	- Interfaz de usuarios
	- Formularios de experiencia
	- Contacto del alumno con el profesor y viceversa
 
- **Fuera de alcance:**
  - Elaborar un diseño visual más elaborado
  - Interfaz de administración de usuarios
  - Contar con acceso a calendarios, clases y estadística 


#### 5. ESPECIFICACIONES FUNCIONALES
  - Sistema de búsqueda diferenciado para distintos roles.
  - Sistema de navegación diferenciado de acuerdo al propósito de cada perfil.
  - Diseño de interfaces e inicio de sesión para distintos roles
  - Encuesta modular para los alumnos 
  - Uso de bases de datos para almacenar la información de los alumnos


#### 6. REQUISITOS NO FUNCIONALES

| **Categoría** | **Requisito** |
| :--- | :--- |
| **Accesibilidad** | La web debe ser navegable mediante teclado y lectores de pantalla. |
| **Seguridad** | La web contará con un sistema de inicio de sesión |
| **Contacto** | La interfaz de profesor y alumno propiciará una comunicación eficaz para el apoyo y asesoría |
| **Recabado de información** | Funcionalidad de las encuestas y envío de los datos particulares para la donación de información destinada a la construcción de material y experiencias de asesoría |


#### 7. ARQUITECTURA DE LA INFORMACIÓN Y UX
- Barra lateral de navegación desplegada mediante un ícono
- Barra de búsqueda
- Interfaz diferenciada por usuario
- Mapa de sitio
- Paleta de colores: azul marino, naranja, dorado, blanco y gris.

#### 8. ESPECIFICACIONES TÉCNICAS

- **Frontend:** HTML para el enmaquetado de nuestra página y CSS para el diseño y uso responsivo de la página.
- **Backend:** PHP para la cuestion logica. 
- **Base de Datos:** MariaDB para guardar datos específicos de los usuarios.
